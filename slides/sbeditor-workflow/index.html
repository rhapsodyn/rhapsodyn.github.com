## SBEditor中项目管理工具使用情况总结
## Content * Git * Redmine * Misc
## Git
### Git客户端 * 推荐CLI工具 * 用惯了GUI（tortoise），不换个口味来一发？ * 其实常用命令也就只有`git pull` `git commit` `git push` `git status` `git diff`而已
### Gitlab * [http://192.168.31.222:8080/](http://192.168.31.222:8080/) * 请大家自行注册并麻烦管理员加入到`unity3d`组下 * http和ssh协议都支持
### Workflow * [http://192.168.31.222:8080/unity3d/sbeditor](http://192.168.31.222:8080/unity3d/sbeditor) * 新建了一个`dev`分支 * 每天的开发过程，我们很不git的把remote都指向了`dev` * 每天下班之前，在网页上发起merge request，最后把`dev`分支merge到了`master` * jenkins每天的build都只从`master`上拉代码
### Workflow * 到每天merge之前，都在做着很svn的事情 * Why not github-way? * conflict解决于无形 * 互相依赖的代码更方便的管理 * 每天只做一次merge * 希望大家尝试其他方式
## Redmine
### Redmine与Gitlab联动 * 可以`commit -m 'close #taskid'` * redmine自动关闭任务或bug单
### 请不要照上一页操作 ![cry](cry.png)
### 因为 * redmine要单项目配置 * **不够智能，commit一旦push了，task就已经关了** * 所以，还是每天下班老老实实去redmine关下任务吧
